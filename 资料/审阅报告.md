\# MSPM0G3507 智能小车航向系统代码审阅报告



\## 一、项目与审阅目标



项目仓库：



```text

https://github.com/Lcxzyt/02\_2UART.git

```



主要硬件：



```text

主控：MSPM0G3507

惯性传感器：MPU6050

磁力计：QMC5883P 或兼容型号

电机驱动：TB6612FNG

```



本次审阅目标不是简单让小车“短时间看起来走直”，而是核对并修复整个航向测量与控制链，使其满足：



1\. 小车启动时捕获的参考方向在任务过程中保持不变；

2\. 小车静止和直线运行时，方向角不会因陀螺仪漂移或磁力计干扰明显偏移；

3\. 去程和回程目标方向在同一航向坐标系中严格相差 180°；

4\. 传感器偶发读取失败后，不得把错误方向自动设成新的目标方向；

5\. 电机启动、停止、正转、反转时，航向输出不能出现不可控跳变；

6\. 所有结论必须根据当前代码和实测日志核对，不能直接照本报告盲目修改。

\# 三、总体审阅结论



当前工程最主要的问题不是“回程目标角没有加 180°”，而是：



> 当前用于控制的航向测量值不够可靠。



当前版本的核心问题包括：



1\. 当前 `Heading.c` 已退化为直接使用磁力计航向；

2\. MPU6050 的陀螺仪 Z 轴没有进入当前航向输出；

3\. 磁力计读取失败没有正确传播到上层；

4\. 磁力计标定只做了硬铁偏移，没有真正应用软铁补偿；

5\. 融合器历史版本虽然存在，但缺少真实陀螺仪零偏校准，并且使用整数保存融合航向；

6\. `HeadingDrive` 在部分异常恢复路径中可能覆盖原目标方向；

7\. 去程和回程的 `+180°` 逻辑本身已经存在，不是主要根因；

8\. 传感器坐标系和车体坐标系是否一致尚未得到严格验证；

9\. 电机、动力线、螺丝、电池和车架产生的磁干扰尚未通过对照实验定量判断。



建议修复方向是：



```text

MPU6050 Z轴角速度短期积分

&#x20;       +

QMC5883P 校准后航向低频纠偏

&#x20;       +

磁数据有效性与干扰门控

&#x20;       +

稳定且不可被自动覆盖的目标方向

```

\# 四、已确认或高度可疑的代码问题



\## P0-1：当前控制使用的 Yaw 没有融合陀螺仪



\### 涉及文件



```text

User/app/Heading.c

User/app/IMUTest.c

User/app/IMU.c

```



\### 当前调用链需要核对



```text

Heading\_UpdateWithDt()

&#x20;   -> IMUTest\_Read()

&#x20;       -> IMU\_GetAttitudeRaw()

&#x20;           -> IMU\_AccelToAngles()

&#x20;           -> IMU\_ComputeHeading()

```



当前 `Heading\_UpdateWithDt(float dt\_sec)` 中，`dt\_sec` 被忽略，最终只是将 `imu.YawDeg` 赋给 `heading\_yaw\_deg`。



而 `IMU\_GetAttitudeRaw()` 中：



```c

IMU\_AccelToAngles(sc, \&att->Roll, \&att->Pitch);

att->Yaw = IMU\_ComputeHeading(sc, att->Roll, att->Pitch);

```



也就是说：



```text

Roll/Pitch：由加速度计计算

Yaw：由磁力计直接计算

GyroZ：未参与 Yaw

```



即使调用现有的 `IMU\_GetAttitudeKF()`，也需要核对它是否只融合了 Roll/Pitch，而 Yaw 仍然直接来自磁力计。



\### 后果



电机磁场、动力线电流、铁磁螺丝和车架造成的磁力计误差，会直接进入方向 PID。



表现可能是：



```text

电机一启动，Yaw 改变

左右电机输出不同时，Yaw 向某一方向偏

小车实际没转，航向角却慢慢变化

同一个物理方向，去程和回程测得的误差不对称

```



\### 建议



重新实现一维航向融合器：



```c

yaw\_predict = yaw\_fused + (gyro\_z\_dps - gyro\_bias\_z) \* dt;

mag\_error   = Wrap180(yaw\_mag - yaw\_predict);



if (mag\_valid \&\& mag\_not\_disturbed) {

&#x20;   yaw\_fused = Wrap360(yaw\_predict + correction\_gain \* mag\_error);

} else {

&#x20;   yaw\_fused = Wrap360(yaw\_predict);

}

```



内部 `yaw\_fused` 必须使用 `float`，不能每个周期转成整数。

\## P0-3：磁力计读取失败未正确传播，上层仍可能认为数据有效



\### 涉及文件



```text

User/bsp/QMC5883L.c

User/app/IMU.c

User/app/IMUTest.c

```



当前 `QMC5883L\_GetData()` 读取失败时，很可能执行：



```c

\*magX = 0;

\*magY = 0;

\*magZ = 0;

return false;

```



但是上层的 `IMU\_ReadScaled()` 或 `IMUTest\_Read()` 没有完整保留本次读取返回值。



最终可能出现：



```text

I2C 读取失败

&#x20;   -> 磁数据变成 0,0,0

&#x20;   -> atan2(0,0) 产生无意义航向

&#x20;   -> IMUTest\_Read() 仍根据“初始化成功”返回 true

&#x20;   -> HeadingDrive 将错误航向用于 PID

```



\### 建议



统一一次采样的数据结构，例如：



```c

typedef struct {

&#x20;   IMU\_RawData raw;

&#x20;   IMU\_ScaledData scaled;



&#x20;   uint8\_t mpu\_valid;

&#x20;   uint8\_t mag\_valid;

&#x20;   uint8\_t mag\_ready;

&#x20;   uint8\_t mag\_overflow;

} IMU\_Sample;

```



使用：



```c

bool IMU\_ReadSample(IMU\_Sample \*sample);

```



明确区分：



```text

传感器初始化成功

本帧读取成功

本帧磁数据已更新

本帧磁数据发生溢出

```



初始化成功不能等同于本帧数据有效。

\## P0-4：异常恢复后可能偷偷覆盖目标方向



\### 涉及文件



```text

User/app/HeadingDrive.c

```



请重点核对以下逻辑：



```c

if (hd\_data.state != HD\_STATE\_RUN) {

&#x20;   hd\_data.target\_yaw = hd\_data.current\_yaw;

&#x20;   hd\_data.state = HD\_STATE\_RUN;

&#x20;   HeadingDrive\_ResetController();

}

```



如果某一帧传感器异常导致状态离开 `RUN`，下一帧恢复后，这段代码可能将当前航向写入 `target\_yaw`。



这会造成：



```text

原目标方向：90°

传感器异常时小车已偏到 98°

传感器恢复后将 98°写成新目标

之后小车不再尝试回到90°

```



\### 修复原则



`target\_yaw` 只能在以下情况下改变：



1\. 用户主动执行锁定当前方向；

2\. 自动任务开始时捕获初始方向；

3\. 自动任务明确设置回程方向；

4\. 上层任务明确调用 `HeadingDrive\_SetTargetYaw()`。



传感器异常恢复、控制器状态切换和 PID 复位都不得自动修改目标角。



建议将：



```c

target\_yaw

```



与：



```c

controller state

sensor state

```



彻底解耦。

\## P1-1：旧融合版本没有真正进行陀螺仪 Z 轴零偏校准



旧版本虽然存在：



```c

gyro\_dps = gyro\_z\_raw / 16.4f;

```



但需要核对 `gyro\_z\_bias` 是否始终被设为 0。



MPU6050 即使静止，GyroZ 原始值通常也不会严格为 0。若不校准：



```text

静止零偏 0.5°/s

60 秒累计约 30°

```



即使磁力计会慢慢拉回，也会形成持续的融合误差和方向振荡。



\### 建议实现



上电后保持小车静止 1～2 秒，采样 200～500 次：



```c

bias = sum(gyro\_z\_dps) / sample\_count;

```



校准期间应满足：



```text

左右电机停止

小车静止

加速度模长接近1g

GyroZ变化不超过阈值

```



若检测到小车在移动，应重新开始校准或报告校准失败。



还可以在长期静止时缓慢更新零偏，但不能在小车转向时学习零偏。

\## P1-2：融合航向不能使用 int16\_t 保存内部状态



旧版 `Heading\_Data` 中的：



```c

int16\_t yaw\_deg;

```



不适合作为积分器内部状态。



20 ms 周期下：



```text

1°/s 对应每帧 0.02°

10°/s 对应每帧 0.2°

```



若每一帧都将结果四舍五入为整数，小于约 0.5° 的增量可能反复丢失。



\### 正确设计



内部使用：



```c

float yaw\_fused\_deg;

float mag\_yaw\_deg;

float gyro\_bias\_z\_dps;

```



只在显示或旧接口处转换：



```c

int16\_t Heading\_GetYawDeg(void)

{

&#x20;   return RoundToInt16(yaw\_fused\_deg);

}

```



控制器也最好使用浮点角度误差，最后再将差速输出转换为整数。

\## P1-3：磁力计标定只应用了硬铁偏移



\### 涉及文件



```text

User/app/CmdDispatch.c

User/app/IMUTest.c

User/app/IMU.c

py/mag\_calibration\_analysis.py

```



当前自动标定大致计算：



```c

offset\_x = (max\_x + min\_x) / 2;

offset\_y = (max\_y + min\_y) / 2;

offset\_z = (max\_z + min\_z) / 2;

```



然后调用：



```c

IMUTest\_SetMagCalibration(

&#x20;   offset\_x, offset\_y, offset\_z,

&#x20;   1.0f, 1.0f, 1.0f

);

```



但 `IMUTest\_SetMagCalibration()` 中很可能直接忽略：



```c

scaleX

scaleY

scaleZ

```



所以目前只做了硬铁偏移，没有修正：



```text

X/Y轴增益不同

XY椭圆旋转

轴间耦合

软磁畸变

```



\### 后果



即使输出能够覆盖 0～360°，也不能说明航向准确。



例如真实每转 45°，输出可能是：



```text

0°

31°

72°

126°

184°

226°

285°

333°

```



它仍覆盖一整圈，但角度严重非线性。



\### 建议



对于主要水平行驶的小车，优先实现二维椭圆校准：



```text

\[mx\_cal]   \[a b] \[mx\_raw - ox]

\[my\_cal] = \[c d] \[my\_raw - oy]

```



简单版本至少需要应用 X/Y 比例：



```c

mx\_cal = (mx\_raw - offset\_x) \* scale\_x;

my\_cal = (my\_raw - offset\_y) \* scale\_y;

```



更好的版本使用二维椭圆拟合得到完整 2×2 修正矩阵。



如果小车存在明显俯仰或需要完整倾斜补偿，再做三维椭球拟合。

\## P1-6：不要只修改 MPU6050 量程而不修改比例系数



当前 MPU6050 很可能配置为：



```c

GYRO\_CONFIG = 0x18;   // ±2000°/s

```



对应比例：



```c

16.4 LSB/(°/s)

```



如果改为 ±500°/s 或 ±1000°/s，必须同步修改换算系数。



建议统一定义：



```c

\#define MPU\_GYRO\_FS\_SEL       1U

\#define MPU\_GYRO\_LSB\_PER\_DPS  65.5f

```



并用同一个宏生成寄存器配置，避免驱动配置与算法标度不一致。



不能只复制一段新的 MPU6050 初始化代码而保留原来的：



```c

\#define GYRO\_SCALE (1.0f / 16.4f)

```

\## P1-7：一次控制周期内可能重复读取两遍传感器



请核对 `IMUTest\_Read()` 是否先执行：



```c

IMU\_ReadRaw(\&raw);

```



随后又执行：



```c

IMU\_ReadScaled(\&sc);

```



而 `IMU\_ReadScaled()` 内部再次读取 MPU6050 和磁力计。



这样会导致：



1\. 每个周期重复进行多次 I2C 通信；

2\. `raw.GyroZ` 和最终用于磁航向计算的数据不一定来自同一时刻；

3\. 增加 I2C 被电机干扰的概率；

4\. 调试日志中的原始数据和最终航向可能不完全对应；

5\. 控制周期执行时间不稳定。



\### 建议



每个 20 ms 周期只采样一次：



```c

IMU\_ReadSample(\&sample);

IMU\_ConvertSample(\&sample);

Heading\_UpdateFromSample(\&sample, dt);

```



原始值、缩放值、磁航向和融合航向必须来自同一帧数据。

\# 五、去程和回程 180°逻辑审阅



\## 当前逻辑



请核对 `User/task/AutoTrackTask.c` 是否已经包含：



```c

auto\_base\_yaw = HeadingDrive\_GetTargetYaw();



auto\_reverse\_yaw =

&#x20;   NormalizeYaw(auto\_base\_yaw + 180);

```



如果存在，则程序定义层面的去程和回程目标已经严格相差 180°。



因此，把角度改写成方向向量：



```c

vx = cos(yaw);

vy = sin(yaw);



reverse\_vx = -vx;

reverse\_vy = -vy;

```



不会从根本上提高精度，因为数学结果仍然等价于：



```c

reverse\_yaw = yaw + 180°;

```



\## 真正的问题



实际方向不相差 180°，通常是因为测量误差具有方向相关性：



```text

真实方向0°，测得5°

真实方向180°，测得171°

```



这两个测量结果只差 166°，尽管程序目标值严格相差 180°。



方向相关误差可能来自：



```text

磁力计软铁畸变

车架或螺丝磁化

电机正反转磁场不同

左右电机电流不同

动力线布局不对称

传感器坐标映射错误

```



\## 正确的往返策略



任务启动时：



```c

forward\_target = fused\_yaw;

reverse\_target = Wrap360(forward\_target + 180.0f);

```



整个任务期间：



```text

forward\_target 不再重新捕获

reverse\_target 不再重新捕获

转弯过程中继续更新 fused\_yaw

进入回程直线时直接使用 reverse\_target

不得重新用当前磁航向执行 snap

```



只有用户重新开始任务时，才能重新捕获新的参考方向。

\# 六、推荐的航向模块结构



建议将航向测量和航向控制彻底分层。



\## 1. IMU 采样层



负责：



```text

读取 MPU6050

读取 QMC5883P

检查通信状态

检查磁力计 DRDY

检查磁力计 OVFL

进行原始数据缩放

应用坐标轴映射

应用磁力计校准矩阵

```



建议接口：



```c

bool IMU\_ReadSample(IMU\_Sample \*sample);

```



\## 2. Heading 航向估计层



负责：




\# 七、推荐的一维航向融合算法



对于水平地面电赛小车，推荐先实现以下算法。



```c

static float yaw\_fused\_deg;

static float gyro\_bias\_z\_dps;

static uint8\_t yaw\_seeded;



void Heading\_Update(const IMU\_Sample \*s, float dt)

{

&#x20;   float gyro\_rate;

&#x20;   float yaw\_predict;



&#x20;   gyro\_rate = s->gyro\_z\_dps - gyro\_bias\_z\_dps;

&#x20;   yaw\_predict = Wrap360(yaw\_fused\_deg + gyro\_rate \* dt);



&#x20;   if (!yaw\_seeded) {

&#x20;       if (s->mag\_valid \&\& !s->mag\_disturbed) {

&#x20;           yaw\_fused\_deg = s->mag\_yaw\_deg;

&#x20;           yaw\_seeded = 1U;

&#x20;       }

&#x20;       return;

&#x20;   }



&#x20;   if (s->mag\_valid \&\& !s->mag\_disturbed) {

&#x20;       float mag\_error =

&#x20;           Wrap180(s->mag\_yaw\_deg - yaw\_predict);



&#x20;       if (fabsf(mag\_error) <= MAG\_ERROR\_GATE\_DEG) {

&#x20;           yaw\_fused\_deg =

&#x20;               Wrap360(yaw\_predict +

&#x20;                       MAG\_CORRECTION\_GAIN \* mag\_error);

&#x20;       } else {

&#x20;           yaw\_fused\_deg = yaw\_predict;

&#x20;       }

&#x20;   } else {

&#x20;       yaw\_fused\_deg = yaw\_predict;

&#x20;   }

}

```



初始参数可先使用：



```c

\#define MAG\_CORRECTION\_GAIN   0.01f

\#define MAG\_ERROR\_GATE\_DEG    20.0f

```



但参数必须根据日志调整。



不要将“转弯时磁航向与预测值差异较大”简单等同于磁干扰。真实快速转向时，只要磁力计和陀螺仪坐标、符号、时间同步正确，两者应该大体一致。



更可靠的磁干扰判定还应检查：



```text

磁场模长是否偏离标定环境

磁数据是否溢出

单帧变化是否不合理

磁航向和陀螺预测差是否突然异常

```

\# 八、磁力计干扰判定建议



校准完成、远离电机干扰时，记录磁场模长参考值：



```c

mag\_norm\_ref = sqrtf(mx \* mx + my \* my + mz \* mz);

```



运行时计算：



```c

ratio = mag\_norm / mag\_norm\_ref;

```



初步可将以下情况视为异常：



```text

ratio < 0.70

ratio > 1.30

状态寄存器出现溢出

数据未Ready

单帧磁向量突变超过阈值

```



阈值必须根据实测调整。



注意：



```text

磁场模长正常不代表方向一定正常；

磁场模长异常则基本可以判定磁数据不可靠。

```

