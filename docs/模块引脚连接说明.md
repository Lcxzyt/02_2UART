# 智能寻迹小车模块引脚连接说明

本文档对应嘉立创天猛星 MSPM0G3507 开发板智能寻迹小车工程。引脚分配以当前工程的 `empty.syscfg`、底层驱动源码及实车接线记录为准。

> 最后核对日期：2026-07-16  
> 主控逻辑电平：3.3V  
> 所有外设、电机驱动和主控必须共地。

## 2. TB6612 电机驱动模块

### 2.1 控制端连接

| TB6612 引脚 | MSPM0G3507 引脚 | 功能 |
| --- | --- | --- |
| PWMA | PA27 | 左电机 PWM 调速 |
| AIN1 | PB6 | 左电机方向控制 1 |
| AIN2 | PB7 | 左电机方向控制 2 |
| PWMB | PA26 | 右电机 PWM 调速 |
| BIN1 | PB8 | 右电机方向控制 1 |
| BIN2 | PB9 | 右电机方向控制 2 |
| STBY | 3.3V或模块高电平使能端 | 必须保持高电平，部分模块板已自带上拉 |
| VCC | 3.3V | 逻辑电源 |
| GND | GND | 与主控、电池负极共地 |
| VM | 电机电源正极 | 按电机和驱动模块额定电压供电，不接主控3.3V |

### 2.2 电机端连接

| 电机 | TB6612 输出 | 说明 |
| --- | --- | --- |
| 左电机 | AO1、AO2 | 对应 A 通道 |
| 右电机 | BO1、BO2 | 对应 B 通道 |

注意：当前程序中左电机使用 `PWMA=PA27`，右电机使用 `PWMB=PA26`。PA26、PA27 不要按数字顺序想当然地对调。

## 3. 左右编码器

| 编码器线 | MSPM0G3507 引脚 | 说明 |
| --- | --- | --- |
| 左编码器 B 相 | PB4 | 正交编码输入 |
| 左编码器 A 相 | PB5 | 正交编码输入 |
| 右编码器 A 相 | PB2 | 正交编码输入 |
| 右编码器 B 相 | PB3 | 正交编码输入 |
| 编码器 VCC | 按编码器额定电压连接 | 输出高电平不得超过3.3V |
| 编码器 GND | GND | 必须与主控共地 |

四路信号在软件中均启用了内部上拉和上升沿、下降沿中断。若编码器输出是5V电平，必须先确认其输出形式和主控引脚耐压；不能确认时应加电平转换或分压保护。

## 4. 八路模拟红外循迹模块

当前工程按“一个模拟输出加三根地址线”的八路复用方式读取，不是八根数字灰度输入。

| 循迹模块/复用器信号 | MSPM0G3507 引脚 | 功能 |
| --- | --- | --- |
| OUT / SIG | PA16 | ADC1-A1模拟采样输入 |
| A0 / S0 | PA17 | 通道选择位0 |
| A1 / S1 | PB17 | 通道选择位1 |
| A2 / S2 | PB18 | 通道选择位2 |
| VCC | 3.3V或模块规定电压 | 需保证OUT最高电压不超过主控模拟输入范围 |
| GND | GND | 与主控共地 |

程序依次选择并采集 `Track[0]` 到 `Track[7]`。当前实测安装方向已经在软件中做过地址和左右方向修正，因此更换模块或改变排线方向后需要重新检查通道顺序及黑白标定值。

## 5. I²C传感器和OLED

OLED、MPU6050和QMC5883P共用同一组I²C总线。

| 模块引脚 | MSPM0G3507 引脚 | 说明 |
| --- | --- | --- |
| SDA | PA0 | I2C0数据线 |
| SCL | PA1 | I2C0时钟线，当前速率100kHz |
| VCC | 建议3.3V | 以模块规格为准 |
| GND | GND | 共地 |

当前软件使用的7位I²C地址如下：

| 设备 | 地址 | 备注 |
| --- | --- | --- |
| OLED | `0x3C` | 固定使用 |
| MPU6050 | `0x68` | AD0应保持低电平；若AD0拉高会变为`0x69`，当前程序不能直接识别 |
| QMC5883P | 优先`0x0D` | 程序还会尝试`0x1A`、`0x2C`并扫描总线 |

接线注意事项：

- SDA、SCL需要上拉到3.3V；多数模块板自带上拉，多个模块并联时应避免总上拉阻值过小。
- MPU6050需要开启旁路模式，QMC5883P才能由主控直接访问；当前程序已经完成该配置。
- 磁力计应远离电机、TB6612、大电流电源线、螺丝和其他铁磁材料，否则航向角会明显漂移。
- OLED刷新和IMU读取共用总线，接触不良可能同时表现为屏幕异常和航向控制失败。

## 6. 蓝牙串口模块

| 蓝牙模块引脚 | MSPM0G3507 引脚 | 说明 |
| --- | --- | --- |
| RX | PA8 / UART1 TX | TX接RX，必须交叉连接 |
| TX | PA9 / UART1 RX | RX接TX，必须交叉连接 |
| VCC | 按模块规格连接 | 信号电平必须兼容3.3V |
| GND | GND | 共地 |

通信参数为 `9600 bit/s`、8位数据、无校验、1位停止位。

## 7. USB/调试串口

| USB转串口端 | MSPM0G3507 引脚 | 说明 |
| --- | --- | --- |
| RX | PA10 / UART0 TX | 接收主控输出 |
| TX | PA11 / UART0 RX | 向主控发送命令 |
| GND | GND | 共地 |

通信参数为 `115200 bit/s`、8位数据、无校验、1位停止位。若开发板已经通过板载调试器连接UART0，不需要重复外接USB转串口模块。

## 8. 按键、LED和蜂鸣器

| 器件 | MSPM0G3507 引脚 | 有效电平 | 功能 |
| --- | --- | --- | --- |
| MENU按键 | PB21 | 低电平 | 切换OLED页面 |
| FUNC按键 | PB16 | 低电平 | 执行当前页面功能 |
| LED | PB22 | 高电平 | 状态和任务提示 |
| 蜂鸣器 | PB23 | 低电平 | 任务节点、完成和故障提示 |

按键输入已经启用内部上拉，外接按键时应接在对应GPIO与GND之间。外接大电流蜂鸣器时不要由GPIO直接驱动，应增加三极管或MOS管驱动电路。

## 9. SWD下载与调试接口

| 调试信号 | MSPM0G3507 引脚 |
| --- | --- |
| SWDIO | PA19 |
| SWCLK | PA20 |
| GND | GND |
| VTREF | 3.3V |

使用板载调试器或外部J-Link时必须连接公共地和目标电压参考。PA19、PA20应保留给下载调试，不建议复用为普通GPIO。

## 10. 上电前检查清单

1. 主控、TB6612、传感器、蓝牙和电池负极已经共地。
2. TB6612的逻辑电源VCC和电机电源VM没有接反，STBY保持高电平。
3. 左电机接AO1/AO2，右电机接BO1/BO2。
4. PWMA接PA27，PWMB接PA26。
5. 编码器左右和A/B相没有接反，输出电平不超过3.3V。
6. 红外模块OUT接PA16，A0/A1/A2分别接PA17/PB17/PB18。
7. I²C总线SDA接PA0、SCL接PA1，并上拉到3.3V。
8. 蓝牙和USB串口均按TX接RX、RX接TX交叉连接。
9. 首次上电时先架空车轮测试方向、编码器符号和停车命令，再放到地面运行。
