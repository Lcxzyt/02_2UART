# 八路数字灰度GPIO改造方案

## 1. 方案状态

- 当前状态：仅完成方案设计，尚未修改代码和SysConfig。
- 目标硬件：嘉立创天猛星MSPM0G3507开发板。
- 目标传感器：具有8路独立数字输出的灰度/红外循迹模块。
- 选定方案：使用GPIOB的8个空闲引脚直接读取高低电平。
- 保留原则：电机、编码器、OLED、IMU、蓝牙、按键、LED、蜂鸣器和SWD引脚均不调整。

## 2. 改造目标

当前八路灰度传感器采用以下接口：

- `PA16`：一路复用模拟输出，使用ADC1-A1采样。
- `PA17`：MUX地址A0。
- `PB17`：MUX地址A1。
- `PB18`：MUX地址A2。

改造后取消模拟复用读取，改为8路数字信号同时接入GPIO。主控每个20ms控制周期读取一次8路电平，根据黑线有效通道计算巡线位图和位置误差。

本次改造不追求中断采样，使用主循环20ms轮询即可满足当前小车的巡线控制周期。

## 3. 最终引脚分配

传感器通道按小车前进方向观察，从物理左侧到物理右侧依次编号为 `S0`～`S7`。

| 物理位置 | 传感器输出 | MSPM0G3507引脚 | GPIO端口位 | 当前工程状态 |
| --- | --- | --- | --- | --- |
| 最左 | S0 / D0 | PB10 | GPIOB bit 10 | 空闲 |
| 左2 | S1 / D1 | PB11 | GPIOB bit 11 | 空闲 |
| 左3 | S2 / D2 | PB12 | GPIOB bit 12 | 空闲 |
| 左4 | S3 / D3 | PB13 | GPIOB bit 13 | 空闲 |
| 右4 | S4 / D4 | PB14 | GPIOB bit 14 | 空闲 |
| 右3 | S5 / D5 | PB15 | GPIOB bit 15 | 空闲 |
| 右2 | S6 / D6 | PB19 | GPIOB bit 19 | 空闲 |
| 最右 | S7 / D7 | PB20 | GPIOB bit 20 | 空闲 |

如果传感器丝印使用 `D1`～`D8`，则按照下表连接：

| 传感器丝印 | MSPM0G3507引脚 |
| --- | --- |
| D1 | PB10 |
| D2 | PB11 |
| D3 | PB12 |
| D4 | PB13 |
| D5 | PB14 |
| D6 | PB15 |
| D7 | PB19 |
| D8 | PB20 |

> 通道左右顺序必须通过实车测试确认。若模块排线方向与安装方向相反，优先在软件通道映射表中反转，不交叉修改8根物理接线。

## 4. 选择该组引脚的原因

1. 8个引脚在当前工程中均未使用。
2. 全部属于GPIOB，可以一次读取GPIOB端口后通过掩码提取状态。
3. 不占用PA19、PA20的SWD下载调试接口。
4. 不影响PA0、PA1上的OLED、MPU6050和QMC5883P共用I²C总线。
5. 不影响电机PWM、方向控制和四路编码器输入。
6. 不影响USB串口、蓝牙串口以及板载按键、LED和蜂鸣器。
7. 原模拟方案使用的PA16、PA17、PB17、PB18可以释放，留作后续扩展。

## 5. 接线要求

### 5.1 基本接线

| 数字灰度模块 | 开发板连接 | 说明 |
| --- | --- | --- |
| VCC | 3.3V，或按模块规格供电 | 必须确认数字输出电平兼容3.3V |
| GND | GND | 与主控、电机驱动共地 |
| S0～S7 | PB10～PB15、PB19、PB20 | 按第3节顺序连接 |

### 5.2 电平要求

- 在没有确认模块输出结构前，应按MSPM0输入只能承受3.3V逻辑电平处理。
- 如果传感器使用5V供电且数字输出高电平接近5V，必须增加电平转换，或者确认模块能够在3.3V供电下可靠工作。
- 如果输出为开漏/开集电极，需要上拉到3.3V，可使用主控内部上拉或外部约4.7kΩ～10kΩ上拉电阻。
- 如果输出为推挽结构，GPIO应配置为普通数字输入，通常不再启用强上拉或下拉。

### 5.3 供电去耦

建议在传感器模块电源入口附近放置：

```text
VCC与GND之间：10µF～47µF + 0.1µF
```

电容应靠近传感器板供电接口，GND线应可靠连接。不要在8路数字输出与GND之间直接并接大电容，以免信号边沿变慢。

## 6. 与当前引脚的关系

切换为数字方案后，以下原模拟循迹引脚不再使用：

| 引脚 | 原功能 | 改造后状态 |
| --- | --- | --- |
| PA16 | ADC1-A1，复用模拟输出 | 释放 |
| PA17 | MUX A0 | 释放 |
| PB17 | MUX A1 | 释放 |
| PB18 | MUX A2 | 释放 |

正式改代码前，不应提前删除这些引脚的现有初始化。模拟传感器仍在使用期间，继续保持当前配置。

## 7. 后续软件改造范围

本节只记录未来实施内容，本次不修改源码。

### 7.1 GPIO初始化

后续在SysConfig中增加8路GPIO输入，或者在 `Tracking.c` 中使用DriverLib初始化：

- PB10
- PB11
- PB12
- PB13
- PB14
- PB15
- PB19
- PB20

优先建议通过SysConfig管理引脚，统一生成端口、PIN和IOMUX宏，避免手写IOMUX编号。

### 7.2 输入极性配置

不同数字灰度模块可能采用不同有效电平，代码应保留统一配置项，例如：

```c
#define TRACK_DIGITAL_ACTIVE_LOW 1U
```

- 黑线输出低电平：配置为 `1U`。
- 黑线输出高电平：配置为 `0U`。

上电后必须先打印一次8路原始电平，在白底和黑线上分别确认极性，不能仅凭模块说明书猜测。

### 7.3 保持Tracking_Data接口兼容

现有 `LineFollow`、`AutoTrackTask`、OLED和串口输出均依赖 `Tracking_Data`。为了减少上层改动，数字输入仍应填充原数据结构：

| 字段 | 数字方案建议值 |
| --- | --- |
| `raw[i]` | GPIO原始电平，取0或1 |
| `norm[i]` | 检测到黑线为1000，否则为0 |
| `filt[i]` | 经数字滤波后为1000或0 |
| `strength` | 8路 `filt[i]` 之和，范围0～8000 |
| `error` | 有效通道位置权重的加权平均 |
| `valid` | GPIO初始化完成且本次读取有效时为1 |

建议继续使用当前位置权重：

```text
S0   S1   S2   S3   S4   S5   S6   S7
-3500 -2500 -1500 -500 +500 +1500 +2500 +3500
```

这样可以保留现有误差正负方向和差速PID结构。

### 7.4 位图定义

建议保持当前位图约定：

```text
bit0 = S0，物理最左
bit1 = S1
...
bit7 = S7，物理最右
```

典型状态：

| 位图 | 含义 |
| --- | --- |
| `0x00` | 全白或完全失线 |
| `0x18` | 中间两路检测到黑线 |
| `0x01` | 黑线位于最左侧 |
| `0x80` | 黑线位于最右侧 |
| `0xFF` | 八路全黑 |

### 7.5 数字滤波

数字比较器在黑白边缘可能抖动。建议第一版采用较轻的滤波，不要引入过大延迟：

- 方案A：连续两次读数一致才更新该位。
- 方案B：每路保存最近3次读数，采用2/3多数表决。

当前控制周期为20ms，如果采用3次多数表决，状态确认可能增加20～40ms延迟。高速巡线时应根据实测决定是否保留滤波。

### 7.6 需要复核的上层逻辑

实施时需要重点复核：

1. `LineFollow_UpdateBits()` 的500阈值是否仍适用。若黑线映射为1000，则可以继续使用。
2. 全白失线逻辑是否仍以 `bits == 0x00` 判断。
3. 全黑逻辑是否仍以 `bits == 0xFF` 判断。
4. `AutoTrackTask` 的黑白确认和 `strength` 阈值是否符合数字数据范围。
5. `x`、`X` 红外调试输出是否要增加数字位图字段说明。
6. 模块的黑线有效电平、通道顺序和小车转向修正方向是否正确。

## 8. 推荐实施顺序

1. 断电完成8路信号、3.3V和GND接线。
2. 用万用表或逻辑分析仪确认输出高电平不超过允许范围。
3. 在SysConfig中配置8路GPIO输入，但暂时不启动电机。
4. 修改 `Tracking.c`，保留 `Tracking_Data` 对外接口。
5. 通过USB串口单次打印8路原始电平。
6. 分别在全白、全黑和单路遮挡条件下确认位图。
7. 架空车轮，低速测试误差正负方向与左右轮修正方向。
8. 在地面以低速测试普通循迹、失线停车和全黑停车。
9. 最后测试自动任务的A/B/C/D节点判断。

## 9. 验收标准

改造完成后至少满足以下条件：

- 白底时位图稳定为 `0x00`。
- 全黑时位图稳定为 `0xFF`。
- 从左到右逐路触发时，位图按bit0到bit7依次变化。
- 中线状态下误差接近0。
- 黑线向左移动时误差为负，向右移动时误差为正。
- USB和蓝牙可以正常输出8路数字状态。
- 普通循迹不会左右修正反向。
- 失线约200ms后能够停车，重新找到线后能够恢复。
- 全黑区域按现有安全策略低速确认后停车。
- 自动任务能够正确识别白区、黑线和终点状态。
- OLED、IMU、蓝牙、编码器和电机控制不受新增GPIO影响。

## 10. 回退方案

建议实施时保留模拟和数字两种后端的编译选项，例如：

```c
#define TRACK_INPUT_MODE_ANALOG  0U
#define TRACK_INPUT_MODE_DIGITAL 1U
#define TRACK_INPUT_MODE TRACK_INPUT_MODE_DIGITAL
```

若数字传感器阈值不稳定或高速效果不理想，可以切回原来的PA16模拟采样和三位MUX地址方案，无需修改 `LineFollow` 和任务层接口。

## 11. 方案结论

当前工程有足够GPIO完成8路数字灰度直读。最终预留PB10～PB15、PB19、PB20作为S0～S7输入，能够避开所有现用模块和调试接口。

正式实施时应优先解决输出电平、有效极性和通道左右顺序，再通过兼容 `Tracking_Data` 的方式接入现有巡线与自动任务逻辑。数字方案接线和读取更直接，但会失去模拟强度信息，因此必须经过低速实车测试后再决定是否替代当前模拟方案。
