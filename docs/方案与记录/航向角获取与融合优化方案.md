# 航向角获取与融合优化方案

> 状态：第一至第三阶段已实现并编译通过，待实车验证  
> 适用工程：嘉立创天猛星 MSPM0G3507 智能寻迹小车  
> 传感器：MPU6050 + QMC5883P  
> 基准周期：20 ms（50 Hz）

## 1. 方案结论

当前小车主要在水平地面运动，最终采用一维航向融合，不直接引入完整的 Madgwick MARG 四元数算法：

```text
MPU6050 GyroZ 积分预测
        +
校准后的 QMC5883P 磁航向慢速纠偏
        +
磁数据有效性及磁干扰门控
        =
供直行控制使用的浮点融合航向
```

工程不额外建立“车头坐标系”，统一以 MPU6050 坐标系为计算基准。车头相对传感器的固定安装偏角，由任务开始时捕获当前航向自动抵消。

已经根据传感器丝印方向确认：

```text
MPU6050 +X = QMC5883P -Y
MPU6050 +Y = QMC5883P +X
MPU6050 +Z = QMC5883P +Z
```

因此，QMC5883P 原始数据转换到 MPU6050 坐标系时采用：

```c
mag_x = -qmc_y;
mag_y =  qmc_x;
mag_z =  qmc_z;
```

以上映射只统一两颗传感器的坐标，不把传感器的某个轴强制定义成车头方向。

## 2. 改造目标

1. 每个控制周期只采集一帧 MPU6050 和 QMC5883P 数据。
2. 原始值、缩放值、姿态角和融合角使用同一时刻的数据。
3. GyroZ 参与航向估计，磁力计不再直接控制电机。
4. 磁力计异常或受扰时，短时间依靠陀螺仪维持航向。
5. 航向估计、目标角和角度误差在内部使用 `float`。
6. 正向与反向目标在任务开始时一次性锁定，并始终严格相差 180°。
7. 传感器异常、控制状态切换和 PID 复位不得修改已经锁定的目标角。
8. 提供足够的调试数据，能够区分传感器、融合、磁干扰和控制器问题。

## 3. 不在本阶段处理的内容

- 不修改寻迹状态机和灰度传感器逻辑。
- 不更换 MPU6050 或 QMC5883P。
- 不直接加入完整 Madgwick/Mahony MARG。
- 不把编码器里程计作为主航向来源。
- 在航向测量稳定之前，不重新整定直行 PID。

编码器差速角速度以后可作为航向一致性检查或短时辅助，但轮胎打滑、轮径误差和轮距误差会使其产生累计偏差，因此不作为本阶段主角度来源。

## 4. 当前实现存在的问题

### 4.1 当前 Yaw 是磁力计直接输出

当前调用链为：

```text
Heading_UpdateWithDt()
  -> IMUTest_Read()
    -> IMU_GetAttitudeRaw()
      -> 加速度计计算 Roll/Pitch
      -> QMC5883P 计算 Yaw
```

`dt_sec` 没有参与航向估计，GyroZ 虽被读取但没有参与 Yaw。电机、电源线或铁磁材料造成的磁航向变化会直接进入方向 PID。

### 4.2 一次更新重复读取传感器

`IMUTest_Read()` 先调用 `IMU_ReadRaw()`，随后调用的 `IMU_ReadScaled()` 又读取一遍传感器。由此可能产生：

- 原始 GyroZ 与最终磁航向不属于同一帧；
- I2C 通信量加倍；
- 控制周期执行时间波动；
- 日志数据和实际控制输入不完全对应。

### 4.3 MPU6050 与 QMC5883P 坐标未对齐

当前倾斜补偿直接把 MPU6050 计算的 Roll/Pitch 作用于 QMC5883P 原始 XYZ，但两颗传感器在板上的方向相差 90°。小车完全水平时主要表现为固定角度偏移；发生坡度、振动、加速或刹车时，会产生额外航向误差。

### 4.4 磁力计只应用了硬铁偏移

现有接口虽然包含 `scaleX/scaleY/scaleZ`，但这些参数未参与计算。当前自动校准只能修正圆心偏移，不能修正 X/Y 增益差、椭圆旋转和软铁畸变。

### 4.5 航向过早整数化

当前 Yaw 在进入 `Heading` 后保存为整数。20 ms 周期内，小于约半度的积分增量可能持续丢失，因此融合器内部必须使用浮点状态。

### 4.6 目标角存在被覆盖的潜在路径

`HeadingDrive` 中存在控制状态不是 `RUN` 时把 `target_yaw` 改成 `current_yaw` 的逻辑。当前失败路径通常会停止控制，因此不一定立即触发，但该逻辑会破坏“目标角只由上层明确设置”的约束，应删除。

## 5. 软件分层设计

### 5.1 传感器驱动层

负责：

- MPU6050 一次突发读取；
- QMC5883P 状态读取和一次突发读取；
- I2C 成功/失败状态；
- `DRDY` 和 `OVFL` 状态；
- 原始数据输出，不负责最终航向计算。

### 5.2 IMU采样与校准层

负责：

- 原始值转物理量；
- QMC5883P 到 MPU6050 的坐标映射；
- GyroZ 零偏修正；
- 磁力计硬铁和软铁修正；
- 生成同一帧的完整样本及有效性标志。

建议的数据结构：

```c
typedef struct {
    int16_t accel_raw[3];
    int16_t gyro_raw[3];
    int16_t mag_raw[3];

    float accel_mps2[3];
    float gyro_dps[3];
    float mag_uT[3];       /* 已映射、已校准 */

    uint8_t mpu_valid;
    uint8_t mag_read_valid;
    uint8_t mag_ready;
    uint8_t mag_overflow;
} IMU_Sample;
```

### 5.3 航向估计层

负责：

- Roll/Pitch 滤波；
- 倾斜补偿磁航向；
- GyroZ 积分预测；
- 磁干扰检测；
- 磁航向慢速纠偏；
- 包角和连续角输出；
- 航向质量状态。

建议的数据结构：

```c
typedef struct {
    float yaw_fused_deg;       /* [0, 360)，控制使用 */
    float yaw_continuous_deg;  /* 可跨越多圈，调试使用 */
    float yaw_mag_deg;
    float gyro_z_dps;
    float gyro_bias_z_dps;
    float mag_norm_uT;
    float mag_norm_ref_uT;

    uint8_t gyro_calibrated;
    uint8_t mag_valid;
    uint8_t mag_disturbed;
    uint8_t ready;
    uint8_t state;
} Heading_Data;
```

### 5.4 航向控制层

负责：

- 保存目标航向；
- 计算最短角度误差；
- 航向 PID；
- 把航向修正量转换为左右轮差速。

目标角与控制状态、传感器状态分离。只有以下接口可以修改目标角：

- 用户主动捕获当前方向；
- 自动任务开始时捕获正向方向；
- 自动任务明确设置反向方向；
- 上层显式调用目标角设置接口。

## 6. 单帧数据处理流程

每20 ms执行一次：

```text
读取 MPU6050 一帧
        |
读取 QMC 状态寄存器
        |
状态允许时读取 QMC 一帧
        |
原始值缩放
        |
QMC轴映射：(-Y, +X, +Z)
        |
应用磁力计校准
        |
计算 Roll/Pitch 和磁航向
        |
GyroZ积分预测
        |
磁数据门控
        |
有效：慢速纠偏；无效：仅陀螺预测
        |
输出浮点融合Yaw和质量状态
```

温度不需要每20 ms读取，可降低到约1 Hz，或只在诊断命令中读取。

## 7. 传感器配置与核验

### 7.1 QMC5883P

已经确认目标器件为 QMC5883P 时，驱动应执行：

1. 优先使用7位地址 `0x2C`；
2. 读取 `CHIPID`，要求为 `0x80`；
3. 按 QMC5883P 官方初始化顺序配置；
4. 官方应用示例要求写 `0x29 = 0x06`，应仅在 QMC5883P 专用驱动中使用；
5. 读取前检查 `DRDY`，读取并处理 `OVFL`；
6. ±8 G 模式使用 `3750 LSB/G`，即：

```c
#define QMC_MAG_UT_PER_LSB (100.0f / 3750.0f)
```

当前 `CR1=0x0F` 按位定义表示 OSR2=1、OSR1=8、ODR=200 Hz、连续模式，配置本身有效，不因报告中的示例值不同而盲目改成其他值。最终可根据噪声日志决定保持200 Hz或降低ODR。

如果需要兼容其他磁力计，不再采用“任意候选地址读到数据就当成QMC”的方式，而应为不同芯片建立明确的地址、CHIPID和寄存器配置档案。

### 7.2 MPU6050

建议参数：

| 项目 | 当前 | 建议初值 |
|---|---:|---:|
| 主循环采样 | 50 Hz | 保持50 Hz |
| 陀螺量程 | ±2000°/s | ±500°/s |
| 陀螺比例 | 16.4 LSB/(°/s) | 65.5 LSB/(°/s) |
| 加速度量程 | ±2 g | 保持±2 g |
| DLPF | 约5 Hz | 约42 Hz，实测后调整 |

量程寄存器和换算比例必须由同一组宏定义，禁止只修改寄存器而不修改比例。

如果实测小车最大旋转速度接近或超过500°/s，则改用±1000°/s，并同步使用32.8 LSB/(°/s)。

## 8. GyroZ零偏校准

上电或首次启用航向功能时：

1. 保持电机关闭；
2. 连续采集约2秒；
3. 检查加速度模长接近1 g；
4. 检查三轴角速度变化未超过静止阈值；
5. 对有效帧的 GyroZ 求平均；
6. 有效样本不足时重新计时，不带着错误零偏开始控制。

```c
gyro_z_dps = gyro_z_raw / gyro_lsb_per_dps;
gyro_z_corrected = gyro_z_dps - gyro_bias_z_dps;
```

校准完成前，航向状态为 `CALIBRATING`，电机保持停止。

可在确认小车长期静止时非常缓慢地更新温漂零偏，但不能在转向或行驶时学习零偏。

## 9. 磁力计校准

### 9.1 第一阶段：二维椭圆校准

小车主要水平行驶，优先使用实际装车姿态进行二维椭圆标定：

```text
[mx_cal]   [a b] [mx - ox]
[my_cal] = [c d] [my - oy]
```

它能够修正：

- X/Y硬铁偏移；
- X/Y灵敏度差异；
- 椭圆旋转；
- 水平面内的轴间耦合。

标定时必须保持整车正常装配状态，包括电池、电机、螺丝和线束。采样应覆盖完整一圈，并同时保存原始散点用于离线检查。

### 9.2 第二阶段：按需升级3D椭球

只有当小车存在明显坡度或倾斜补偿误差仍然突出时，才升级为3D椭球校准。3D标定要求人工改变模块姿态、覆盖足够多的球面方向，仅靠小车水平原地旋转无法得到可靠的Z轴和三维软铁参数。

### 9.3 参数保存

最终需要保存：

```text
校准版本和有效标志
硬铁偏移
二维2x2矩阵或三维3x3矩阵
磁场参考模长
校验值
```

参数保存到Flash的具体区域和升级兼容方式，在实施前单独确认，避免与程序或其他持久化数据冲突。

## 10. 航向融合算法

### 10.1 角度工具函数

```c
static float Wrap360f(float angle)
{
    while (angle < 0.0f) angle += 360.0f;
    while (angle >= 360.0f) angle -= 360.0f;
    return angle;
}

static float Wrap180f(float angle)
{
    while (angle > 180.0f) angle -= 360.0f;
    while (angle < -180.0f) angle += 360.0f;
    return angle;
}
```

### 10.2 融合更新

```c
gyro_rate = gyro_z_dps - gyro_bias_z_dps;
gyro_rate *= gyro_z_sign;

yaw_predict = Wrap360f(yaw_fused_deg + gyro_rate * dt_sec);

if (mag_valid && !mag_disturbed) {
    mag_error = Wrap180f(yaw_mag_deg - yaw_predict);
    correction_gain = 1.0f - expf(-dt_sec / correction_tau_sec);
    yaw_fused_deg = Wrap360f(yaw_predict + correction_gain * mag_error);
} else {
    yaw_fused_deg = yaw_predict;
}
```

使用时间常数计算纠偏增益，使融合强度不依赖控制周期是否恰好为20 ms。

建议初值：

| 参数 | 初值范围 | 说明 |
|---|---:|---|
| `correction_tau_sec` | 2～5 s | 越大越信任陀螺仪 |
| 单周期纠偏增益 | 约0.004～0.010 | 20 ms周期下由时间常数计算 |
| 航向创新门限 | 20°～30° | 超过时暂停磁纠偏 |
| 磁场模长门限 | 参考值±25% | 初始值，必须按日志调整 |
| 有效 `dt` | 1～200 ms | 越界时进入异常处理或限幅 |

### 10.3 初始化与重新捕获

融合器第一次得到连续多帧有效磁数据时，用磁航向初始化 `yaw_fused_deg`。初始化后，磁力计只能慢速纠偏，不能每帧直接覆盖融合角。

磁干扰解除后，不立即强制跳到磁航向；先要求磁数据连续稳定，再恢复慢速纠偏。若需要大角度重新对准，只能由明确的用户命令执行。

## 11. 磁数据门控

磁数据允许参与纠偏必须满足：

```text
I2C读取成功
CHIPID和驱动状态正常
DRDY = 1
OVFL = 0
磁场模长在允许范围内
磁航向与陀螺预测创新量在允许范围内
磁向量单帧变化未超过实测阈值
```

推荐判定：

```c
mag_norm = sqrtf(mx * mx + my * my + mz * mz);
norm_ratio = mag_norm / mag_norm_ref;
mag_error = Wrap180f(yaw_mag_deg - yaw_predict);
```

初始门限：

```text
0.75 <= norm_ratio <= 1.25
abs(mag_error) <= 25°
```

门限必须根据实车日志调整。磁场模长正常不代表方向一定正常，但磁场模长明显异常时可以确定该帧不可靠。

状态处理原则：

- 单帧QMC无新数据：继续陀螺预测，不清零航向；
- 短时磁干扰：标记降级，继续陀螺预测；
- 长时间磁数据无效：报告降级状态，是否减速或停车由实测漂移决定；
- MPU6050读取失败：航向不可继续预测，立即停止直行控制；
- 任何异常都不得修改已经保存的目标角。

## 12. 正向与反向目标

任务启动并确认融合航向有效后，只执行一次：

```c
forward_target_deg = yaw_fused_deg;
reverse_target_deg = Wrap360f(forward_target_deg + 180.0f);
```

任务期间：

- A-B直线使用 `forward_target_deg`；
- C-D直线使用 `reverse_target_deg`；
- B-C、D-A曲线期间继续更新融合航向；
- 进入下一段直线时不得重新捕获磁航向；
- 暂停、恢复、传感器异常和PID复位不得修改目标角。

控制误差统一使用：

```c
heading_error_deg = Wrap180f(target_yaw_deg - yaw_fused_deg);
```

角度 `+180°` 与参考向量取反在数学上完全等价。当前工程保留 `+180°` 方案，结构更简单。

## 13. 调试输出

新增航向CSV日志，建议每20 ms或50 ms输出一行：

```text
HDG,time_ms,
gyro_z_raw,gyro_z_dps,gyro_bias_z,
mag_raw_x,mag_raw_y,mag_raw_z,
mag_map_x,mag_map_y,mag_map_z,
mag_cal_x,mag_cal_y,mag_cal_z,
mag_norm,roll,pitch,mag_ready,mag_overflow,mag_disturbed,
mag_yaw,fused_yaw,
target_yaw,heading_error,
left_target,right_target
```

日志应支持关闭，避免正常运行时串口发送阻塞控制循环。

## 15. 分阶段实施顺序

### 第一阶段：修复数据链

1. QMC5883P严格识别；
2. 增加状态受检读取；
3. 单周期单帧采样；
4. 应用 `(-Y, +X, +Z)` 映射；
5. 增加完整日志；
6. 编译并验证原始数据。

### 第二阶段：实现可靠融合

1. 增加GyroZ静止零偏校准；
2. 航向内部状态改为浮点；
3. GyroZ积分预测；
4. 磁航向低频纠偏；
5. 增加磁场模长、状态和创新量门控；
6. 验证顺/逆时针符号一致性。

### 第三阶段：修复目标生命周期

1. 删除状态恢复时修改目标角的逻辑；
2. 自动任务开始时一次捕获正向目标；
3. 一次计算反向目标；
4. 验证整个任务中目标值不变。

### 第四阶段：完善磁校准

1. 导出水平旋转散点；
2. 完成二维椭圆拟合；
3. 应用2×2软铁矩阵；
4. 保存并加载校准参数；
5. 根据坡度测试决定是否升级3D标定。

### 第五阶段：控制器调参

在航向测量通过验收后，再调整：

```text
航向Kp/Ki/Kd
差速输出上限
```

## 16. 实车验证与验收

### 16.1 坐标与符号测试

1. 小车水平静止，记录当前磁航向；

### 16.2 静止漂移测试

条件：电机关闭，小车静止60秒。

记录：

- GyroZ零偏和标准差；
- 纯陀螺积分漂移；
- 融合航向峰峰值；
- 磁门控状态。

### 16.3 0°/180°重复性测试

在地面标记两个严格相反方向，往返至少5次，统计：

- A方向均值、极差；
- B方向均值、极差；
- `Wrap180(B-A)` 与180°的误差；
- 回到A方向后的重复误差。

### 16.4 电机磁干扰测试

小车架空并保持车体不动，依次测试：

```text
电机关闭
双电机同速正转
双电机同速反转
仅左电机转
仅右电机转
左右电机不同速度
```

比较磁向量、磁场模长、磁航向和融合航向。如果车体不动但磁向量随PWM明显变化，应优先调整磁力计位置和动力线布置，软件只能拒绝干扰，不能恢复已经被改变的真实地磁方向。

### 16.5 往返目标锁定测试

任务启动时打印正向和反向目标，整个任务期间确认：

```text
forward_target保持不变
reverse_target保持不变
两者始终相差180°
传感器异常及恢复不改变目标
```

### 16.6 直线控制对比

分别测试：

1. 仅左右轮速度闭环；
2. 当前磁航向直出；
3. 纯GyroZ短时积分；
4. GyroZ与磁航向门控融合。

比较终点横向偏差、最大航向误差、航向均方误差以及左右轮差速输出。

### 19.1 2026-07-19 水平磁标定结果

使用 `C` 命令采集静止电机条件下的水平整圈数据，共313个样本。QMC原生坐标的原始范围为：

```text
X: -475 ... 50 LSB
Y:  986 ... 1497 LSB
Z:  635 ... 679 LSB
```

二维椭圆拟合得到：

```text
硬铁中心: X=-216.835 LSB, Y=1240.726 LSB
等效偏移: X=-5.7823 uT, Y=33.0860 uT

软铁矩阵:
[ 0.980118  0.011096 ]
[ 0.011096  1.020961 ]
```

补偿后水平半径的标准差约为均值的2.1%，优于简单min/max缩放的约2.7%。该结果已作为本车默认参数写入 `IMU.c`，补偿顺序为“QMC原生坐标去偏置和二维矩阵校正，再映射到MPU坐标”。

本次仅进行了水平旋转，Z轴取值范围很窄，不能据此可靠估计Z轴硬铁和三维软铁参数。因此Z偏移暂保持0，二维矩阵也只作用于XY；车辆大倾角运行前仍需补做三维标定。

### 19.2 标定后首轮航向复测

二维补偿烧录后的日志结果：

- 142帧磁数据全部读取有效，无溢出，`mag_disturbed` 全程为0；
- 磁场模长范围约 `18.33~19.42 uT`，标准差约 `0.18 uT`；
- 去程磁航向约由 `256.7°` 变到 `152.6°`，回程后约为 `259.9°`，闭合误差约 `3.2°`；
- 静止时校正后GyroZ接近 `0°/s`；
- 磁航向已能随实际转动近似一比一变化，二维标定达到预期。

`hi` 时的锁定角为 `279°`，开始 `y` 测试时约为 `256~257°`。经确认，两条命令之间人为移动过设备，因此这约22°的差值属于真实姿态变化，不是静止漂移或初始化故障，不据此修改航向初始化逻辑。

### 19.3 四方向倾斜测试

手持设备依次向四个方向倾斜，峰值姿态约为：

```text
Roll  -15.4°
Roll  +37.2°
Pitch +39.2°
Pitch -54.0°
```

测试中IMU和磁力计读取始终有效且无溢出。正Roll和正Pitch产生约 `-95~-103°` 的错误磁航向，被25°创新门拒绝；负Roll和负Pitch产生约 `+20~27°` 的渐进错误，其中一部分能够缓慢绕过创新门并把融合角带偏。这说明二维XY标定在大倾角下不能依赖未校准的QMC Z轴完成可靠倾斜补偿。

现阶段增加二维标定适用范围门控：首次稳定航向同时记录车体正常水平姿态；相对该姿态的Roll/Pitch合成倾角超过12°时，将磁数据标为受扰并暂停磁纠偏，只用GyroZ短时传播；恢复到12°以内后自动重新接受磁航向。该策略适合平面寻迹小车，但若车辆需要长期通过大于12°的坡道，必须升级为三维磁标定，不能长期依赖GyroZ积分。

门控烧录后的四方向复测中，四段大倾角均正确触发，区间约为 `7.8~10.6 s`、`14.3~16.1 s`、`19.0~22.3 s` 和 `25.0~28.4 s`。门控区间内融合角分别只变化约 `+0.26°`、`-0.83°`、`-0.64°` 和 `-0.01°`，与同期GyroZ积分方向和量级一致，未再跟随最大约90°的错误磁航向。复测通过，保留12°门限。
